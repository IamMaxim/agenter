## Purpose

This runbook refreshes the Stage 0 drift guard for the Codex app-server to
`uap/2` adapter plan. The guard compares Agenter's explicit coverage fixture
against the current Codex app-server protocol source in:

- `tmp/codex/codex-rs/app-server-protocol/src/protocol/common.rs`
- `tmp/codex/codex-rs/app-server-protocol/src/protocol/v2.rs`

It covers:

- `ClientRequest` methods generated by `client_request_definitions!`
- `ServerRequest` methods generated by `server_request_definitions!`
- `ServerNotification` methods generated by `server_notification_definitions!`
- `ThreadItem` enum variants

## Refresh After Updating `tmp/codex`

1. Record the new Codex revision:

   ```sh
   git -C tmp/codex rev-parse HEAD
   ```

2. Confirm the source definitions are still in the expected files:

   ```sh
   rg -n "client_request_definitions|server_request_definitions|server_notification_definitions|pub enum ThreadItem" tmp/codex/codex-rs/app-server-protocol/src/protocol
   ```

3. Run the drift guard:

   ```sh
   cargo test -p agenter-runner codex_protocol_coverage
   ```

4. If the test reports `missing coverage rows`, inspect the new Codex variant or
   method in `tmp/codex`, then add a `CoverageRow` to
   `crates/agenter-runner/tests/codex_protocol_coverage.rs` with the intended
   Agenter mapping.

5. If the test reports `stale coverage rows not found in source`, confirm
   whether Codex renamed or removed the method or variant. Update the coverage
   row and the active plan mapping table together so Stage 1+ workers see the
   same contract.

6. Update the revision and refresh date in this runbook.

7. Run hygiene:

   ```sh
   git diff --check
   ```

## Expected Failure Shape

When Codex adds a known app-server method or `ThreadItem` variant without an
Agenter coverage row, the test fails with a message like:

```text
ClientRequest coverage drifted from tmp/codex app-server source.
missing coverage rows: new/method
```

When a coverage row remains after Codex removes or renames a method, the test
reports:

```text
stale coverage rows not found in source: old/method
```

The test also contains a small fixture unit,
`codex_protocol_coverage_reports_unlisted_source_variants`, that proves
unlisted source variants are reported as missing coverage.

## Troubleshooting

- If `cargo test -p agenter-runner codex_protocol_coverage` fails before running
  this test because unrelated runner code does not compile, fix or coordinate
  with the owner of those in-flight changes first. The drift guard is an
  integration test and Cargo must compile the package before it can run.
- If Codex changes the macro syntax substantially, keep the parser simple but
  source-based. The goal is to catch drift in the current checked-out protocol
  source, not to depend on generated TypeScript or runtime reflection.
